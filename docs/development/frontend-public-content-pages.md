# Публичные SSR-страницы контента

Nuxt-страницы публичного контента построены как SSR-first страницы и получают данные из Laravel JSON API.

## Страницы

- `/categories` — список групп категорий и ссылки на будущий мастер подачи обращения с выбранной категорией.
- `/news` — индекс новостей.
- `/news/[slug]` — детальная страница новости.
- `/appeals` — публичный реестр обращений с фильтрами.
- `/appeals/[slug]` — публичная карточка обращения с таймлайном, материалами и комментариями-превью.

## SEO

SEO-данные приходят из API DTO в поле `seo` и применяются через `usePageSeo()`.

В `nuxt.config.ts` добавлены `routeRules`:

- базовые статические страницы prerender;
- публичные контентные страницы `/categories`, `/news/**`, `/appeals/**` через SWR 300 секунд.

## API client

Страницы используют `useApi()`:

- на сервере Nuxt берёт `runtimeConfig.apiInternalBase`;
- в браузере берёт `runtimeConfig.public.apiBase`;
- прямые URL вида `/api/v1/...` в компонентах не используются.

Страница `/categories` получает DTO через `useCategoriesContent()`, который оборачивает `GET /api/v1/categories`. Если в БД нет активных групп с активными категориями, страница остаётся SSR-доступной и показывает пустое состояние вместо ошибки.

## Навигация

Главная страница, header и footer переведены на реальные маршруты:

- `/categories`
- `/news`
- `/appeals`
- `/news/legal-support-consultations`
- `/appeals/road-pits-after-repair`

Ссылки категорий ведут на будущий маршрут `/appeal/new?category=<slug>`.

## Категории в форме обращения

Мастер подачи обращения `/appeal/new` получает категории через `GET /api/v1/categories` и строит выбор темы из API-справочника. Локальный frontend-список категорий не используется, чтобы `/categories`, главная страница и форма не расходились по slug-ам.

Если в query-параметре `category` передан неизвестный slug, форма выбирает первую категорию из API-ответа.
