# Исправить наложение пунктов навигации в desktop-хедере

## Context

В общем `AppHeader` после авторизации справа показываются кнопки `Админка` или `Кабинет`, `Подать обращение` и `Выйти`. При текущей ширине shell-контейнера desktop-навигации не хватает места, поэтому первый пункт меню заезжает на логотип.

## Goal

Сделать desktop-хедер устойчивым к авторизованному состоянию: логотип, навигация и правые действия не должны накладываться друг на друга.

## Non-goals

- Не менять структуру `AppHeader.vue`.
- Не менять auth-flow.
- Не трогать исходную папку `static/`.
- Не менять backend API.

## Backend changes

Backend не меняется.

## Frontend changes

- Обновить CSS shell-хедера в `frontend/app/assets/styles/main.css`.
- Использовать более широкую shell-полосу для header/footer chrome.
- Переключать полный хедер в мобильное меню на ширинах, где desktop-набор ссылок и кнопок не помещается.

## API contract

API-контракт не меняется.

## SEO impact

SEO не меняется. Правка затрагивает только визуальную раскладку общего хедера.

## Tests

- `pnpm typecheck`
- `pnpm lint`
- Визуальная проверка хедера на desktop-ширине с авторизованным состоянием.

## Acceptance criteria

- В desktop-хедере пункт `О нас` не накладывается на логотип.
- В авторизованном состоянии помещаются `Админка` или `Кабинет`, `Подать обращение`, `Выйти`.
- На промежуточных ширинах хедер переходит в hamburger-меню до появления наложений.
- `static/` не изменяется.

## Postman

Не требуется.
