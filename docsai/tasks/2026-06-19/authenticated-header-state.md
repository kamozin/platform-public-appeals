# Состояние шапки для авторизованного пользователя

## Context

На страницах после входа в систему шапка продолжала показывать кнопку `Войти`.
Для авторизованного пользователя это неверное состояние UI: пользователь уже находится в системе и должен видеть переход в личный кабинет или админку.

## Goal

Сделать шапку зависимой от состояния авторизации:

- гостю показывать `Войти`;
- авторизованному пользователю показывать `Кабинет`;
- admin-пользователю показывать `Админка`;
- авторизованному пользователю показывать компактную кнопку `Выйти`;
- сохранить кнопку `Подать обращение`.

## Non-goals

- Не менять backend auth API.
- Не добавлять dropdown-меню профиля.
- Не менять структуру личного кабинета.

## Backend changes

Backend не меняется.
Используется существующий `GET /api/v1/auth/me`, который возвращает `isAdmin`.

## Frontend changes

- Обновить `frontend/app/components/layout/AppHeader.vue`.
- Добавить стили авторизованного состояния в `frontend/app/assets/styles/main.css`.

## API contract

Контракт не меняется.
Frontend использует существующий `useAuth()`:

- `token`;
- `user`;
- `fetchMe()`;
- `logout()`.

## SEO impact

Нет влияния на публичные SEO-страницы.
Изменяется только клиентское состояние шапки после hydration.

## Tests

- `docker compose exec -T nuxt pnpm typecheck`
- `docker compose exec -T nuxt pnpm lint`

## Acceptance criteria

- Гость видит кнопку `Войти`.
- Пользователь с token видит `Кабинет` вместо `Войти`.
- Admin после `auth/me` видит `Админка`.
- Кнопка `Выйти` вызывает существующий logout flow.
- В мобильном меню состояние совпадает с desktop.

## Postman

Не требуется.
