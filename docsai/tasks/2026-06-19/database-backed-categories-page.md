# Подключить страницу категорий к данным из БД

## Context

Публичный endpoint `GET /api/v1/categories` уже читает группы и категории из таблиц `category_groups` и `categories`. Страница `/categories` должна явно работать через этот API-контракт, корректно переживать пустой опубликованный справочник и не расходиться с главной страницей и формой обращения.

## Goal

Закрепить работу `/categories` от данных БД:

- вынести получение данных страницы в отдельный frontend composable;
- показывать пустое состояние, если в БД нет активных опубликованных категорий;
- скрывать публично активные группы без активных категорий;
- покрыть DB-фильтрацию и порядок тестом.

## Non-goals

- Не менять публичный URL `/categories`.
- Не менять формат успешного ответа `GET /api/v1/categories`.
- Не добавлять новые endpoints.
- Не менять admin UI.
- Не редактировать `static/`.

## Backend changes

- Уточнить выборку публичных групп категорий: отдавать только активные группы, у которых есть активные категории.
- Оставить чтение категорий из БД через `PublicContentService`.
- Добавить feature-тест, который создаёт активные и неактивные DB-записи и проверяет публичный ответ.

## Frontend changes

- Добавить `frontend/app/composables/useCategoriesContent.ts`.
- Перевести `frontend/app/pages/categories.vue` на composable.
- Добавить пустое состояние для случая, когда активных категорий нет.

## API contract

Endpoint сохраняется:

- `GET /api/v1/categories`

Успешный ответ остаётся:

```json
{
  "data": {
    "groups": [],
    "seo": {}
  }
}
```

## SEO impact

Страница `/categories` остаётся SSR/SWR и индексируемой. SEO DTO продолжает приходить из API.

## Tests

- `php artisan test tests/Feature/Api/CategoriesEndpointTest.php`
- `corepack pnpm typecheck` во frontend, если окружение доступно.
- `git diff --check` по затронутым файлам.

## Acceptance criteria

- `/categories` получает справочник через общий API client.
- Endpoint отдаёт только активные группы с активными категориями.
- Неактивные категории и группы не попадают в публичный ответ.
- При пустом справочнике страница не падает и показывает понятное состояние.
- Документация на русском обновлена.

## Postman

Postman collection не меняется, потому что endpoint и контракт запроса не меняются.
