# Переделать визуальный стиль круглых иконок и бейджей во frontend

## Context

В личном кабинете и на других frontend-экранах используются круглые светло-синие контейнеры для иконок и рядом стоящие статусные бейджи. На примере списка уведомлений бейдж `прочитано` визуально конфликтует с круглой иконкой: элементы выглядят тяжеловато, плохо разделяются и создают ощущение наложения.

Иконки в Nuxt уже централизованы через `LayoutAppIcon` и SVG-sprite, поэтому задачу нужно решать не заменой отдельных SVG, а единым UI-паттерном для icon badge / status badge.

## Goal

Обновить визуальный стиль подобных иконок и бейджей по всему Nuxt frontend так, чтобы:

- круглые контейнеры стали аккуратнее и современнее;
- статусные бейджи не конфликтовали с иконками;
- состояния `новое`, `прочитано`, `активно`, `готово`, `заблокировано` и похожие выглядели единообразно;
- интерфейс остался плотным и рабочим, без декоративного перегруза.

## Non-goals

- Не менять Laravel API.
- Не менять API-контракты.
- Не редактировать `static/`.
- Не заменять весь SVG-sprite без необходимости.
- Не делать полный редизайн личного кабинета или публичных страниц.
- Не добавлять новую UI-библиотеку.

## Backend changes

Не требуются.

## Frontend changes

Планируемые изменения:

- Ввести единый CSS-паттерн для маленьких иконок в контейнере: размер, радиус, фон, border, цвет SVG, hover/focus если элемент интерактивный.
- Ввести единый CSS-паттерн для статусных бейджей: `blue`, `green`, `gray`, `orange`, `red` с компактной высотой, хорошим контрастом и без наложения на соседние элементы.
- Обновить список уведомлений в `frontend/app/pages/dashboard.vue`: отделить иконку события от статуса чтения, сделать бейджи `новое` и `прочитано` визуально легче.
- Проверить и привести к новому стилю похожие места в `frontend/app/assets/styles/main.css`: карточки статистики, строки активности, достижения, уведомления, admin icon buttons, category icon blocks, ready/status badges.
- Сохранить использование `LayoutAppIcon`; при необходимости добавить только недостающие символы в `AppSvgSprite.vue`.

## API contract

Не меняется.

## SEO impact

Нет влияния на SSR/SEO, так как задача касается визуального оформления существующих frontend-элементов.

## Tests

Проверки:

- `pnpm lint` во `frontend/`, если команда доступна в проекте.
- `pnpm build` или существующая frontend-проверка, если она уже используется.
- Визуальная проверка личного кабинета, уведомлений, категорий, формы обращения и админки.
- Проверка мобильной ширины: бейджи и текст не должны вылезать из карточек.

## Acceptance criteria

- В списке уведомлений `прочитано` больше не выглядит как наложенная круглая иконка.
- Все похожие icon badge / status badge элементы во frontend выглядят единообразно.
- Текст бейджей помещается на desktop и mobile.
- `static/` не изменён.
- Backend-файлы не затронуты.
- Документация-отчёт создана после реализации.

## Postman

Не требуется: API endpoint-ы не меняются.
