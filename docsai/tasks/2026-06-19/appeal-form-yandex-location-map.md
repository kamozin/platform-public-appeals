# Яндекс.Карта для места происшествия в форме обращения

## Context

В форме `/appeal/new` блок "Место происшествия" сейчас использует статическую декоративную карту `.map-mock`. Пользователю нужно выбирать место на реальной Яндекс.Карте и синхронизировать выбранный адрес с текущим полем формы.

Страница `/appeal/new` является workflow-страницей, работает как client-only route и не индексируется.

## Goal

Заменить статический макет карты на интерактивную Яндекс.Карту:

- ввод адреса двигает карту к найденной точке;
- клик по карте или геолокация браузера обновляют адрес в поле;
- маркер показывает выбранную точку;
- при отсутствии API-ключа компонент не ломает форму и показывает fallback-состояние.

## Non-goals

- Не менять backend API.
- Не менять формат черновика обращения.
- Не сохранять координаты в backend в рамках этой задачи.
- Не редактировать `static/`.
- Не добавлять Laravel UI / Blade / Inertia / Livewire.

## Backend changes

Backend не изменяется.

## Frontend changes

- Добавить клиентский компонент карты на базе Yandex Maps JavaScript API 2.1.
- Подключать карту в браузере через публичный runtime config `yandexMapsApiKey`.
- Заменить `.map-mock` на компонент карты в `frontend/app/pages/appeal/new.vue`.
- Добавить стили контейнера карты, loading/error/fallback-состояний и адаптивную высоту.
- Добавить env-переменную `NUXT_PUBLIC_YANDEX_MAPS_API_KEY` в локальную конфигурацию.

## API contract

Контракт API не меняется.

Форма по-прежнему отправляет в `location` человекочитаемую строку адреса через существующие endpoints:

- `POST /api/v1/appeal-drafts`
- `PATCH /api/v1/appeal-drafts/{id}`
- `POST /api/v1/appeal-drafts/{id}/submit`

Координаты не отправляются и не сохраняются.

## SEO impact

SEO-изменений нет. `/appeal/new` остаётся `noindex` и `ssr: false`.

## Acceptance criteria

- [x] В блоке "Место происшествия" отображается интерактивная Яндекс.Карта вместо статического мокапа.
- [x] Адрес из поля формы геокодируется и перемещает маркер на карте.
- [x] Клик по карте обновляет маркер и адрес через обратное геокодирование.
- [x] Кнопка текущей геолокации обновляет карту и адрес, если браузер разрешил доступ.
- [x] Без API-ключа форма остаётся рабочей и показывает понятное fallback-состояние.
- [x] Backend и Postman collection не меняются.

## Postman

Postman collection не требуется обновлять, потому что API endpoints и payload contract не меняются.
