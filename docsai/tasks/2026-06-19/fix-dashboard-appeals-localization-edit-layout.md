# Правка карточек обращений в личном кабинете

## Context

В личном кабинете в списке «Мои обращения» технические значения `zhkh` и `pending_moderation` выводятся напрямую. Карточка плохо выравнивается при длинной дате, а переход к редактированию существующего черновика или обращения на модерации отсутствует.

## Goal

Показывать в личном кабинете только русские пользовательские подписи, добавить корректный переход к редактированию существующего черновика или обращения на модерации и выровнять карточку обращения.

## Non-goals

Не переносим публичную карточку обращения в отдельную модель публикации. Не меняем исходную папку `static/`. Не добавляем Blade UI или Laravel web UI.

## Backend changes

Разрешить обновление содержимого и вложений для обращения со статусом `pending_moderation`. Повторная отправка остаётся доступной только для статуса `draft`.

## Frontend changes

Добавить русские подписи для статусов, категорий и служебных значений. Добавить ссылку редактирования с параметром `draft`. На странице подачи обращения загрузить существующий draft по `id` и заполнить форму.

## API contract

`PATCH /api/v1/appeal-drafts/{id}` и endpoints вложений допускают статусы `draft` и `pending_moderation`. `POST /api/v1/appeal-drafts/{id}/submit` по-прежнему допускает только `draft`.

## SEO impact

Нет. Личный кабинет и форма подачи закрыты от индексации через `useNoindexSeo()`.

## Tests

Обновить feature test для редактирования обращения на модерации и запрета повторной отправки.

## Acceptance criteria

- В карточках личного кабинета нет английских технических значений.
- Черновик открывается на редактирование по своему `id`.
- Обращение на модерации можно открыть и сохранить без повторной отправки.
- Карточка обращения ровно выравнивает категорию, статус, текст, дату и действие.

## Postman

Новые endpoints не добавляются. Контракт существующего `PATCH /api/v1/appeal-drafts/{id}` меняется только по допустимому статусу.
