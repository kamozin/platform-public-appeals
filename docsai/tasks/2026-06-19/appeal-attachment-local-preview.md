# Предпросмотр вложений в форме обращения

## Context

В форме подачи обращения на странице `/appeal/new` уже есть загрузка вложений через существующий API черновиков:

- `POST /api/v1/appeal-drafts/{id}/attachments`;
- `DELETE /api/v1/appeal-drafts/{id}/attachments/{attachment}`.

После загрузки frontend показывает список файлов с типовой заглушкой "Фото", "Видео" или "Документ". API сейчас возвращает только метаданные вложения: `id`, `kind`, `originalName`, `mimeType`, `size`. URL файла или превью в контракте нет.

## Goal

Сделать локальный предпросмотр вложений в текущей сессии формы:

- показывать миниатюру изображения вместо заглушки для JPG/PNG/WEBP;
- показывать видеопревью/постер для MP4/MOV, когда браузер может отрисовать выбранный файл;
- оставлять аккуратную карточку для PDF, но давать открыть PDF в предпросмотре;
- добавить действие "Просмотреть" для вложений с доступным локальным preview;
- открыть preview в модальном окне без перехода со страницы;
- корректно освобождать `Object URL` при удалении файла и уходе со страницы.

## Non-goals

- Не менять папку `static/`.
- Не добавлять публичные URL к приватным вложениям.
- Не делать предпросмотр уже загруженных вложений после перезагрузки страницы или открытия черновика на другом устройстве.
- Не добавлять конвертацию PDF в изображение и генерацию серверных thumbnails.
- Не менять лимиты, типы файлов и правила валидации.
- Не менять процесс submit/moderation.

Постоянный предпросмотр после перезагрузки нужно делать отдельной задачей через защищённый file-preview/download contract.

## Backend changes

Backend не меняется в этой задаче.

Причина: текущий API-only контракт не отдаёт файл или preview URL, а приватные вложения нельзя просто открыть публичной ссылкой. Локальный preview можно сделать на frontend через `URL.createObjectURL(file)` сразу после выбора файла пользователем.

## Frontend changes

Изменить `frontend/app/pages/appeal/new.vue`:

- добавить локальное состояние preview по `attachment.id`;
- при загрузке файла создавать `URL.createObjectURL(file)`;
- после успешного ответа upload API привязывать preview к `response.data.id`;
- для изображений в списке выводить `<img>` внутри `.file-preview`;
- для видео выводить `<video muted preload="metadata">` или оставить типовую карточку, если preview недоступен;
- для PDF показывать кнопку предпросмотра и открывать файл в `<iframe>`/`object` внутри модального окна;
- добавить обработчики открытия/закрытия preview modal;
- при удалении вложения отзывать `URL.revokeObjectURL`;
- в `onBeforeUnmount` отзывать все созданные object URLs.

Изменить `frontend/app/assets/styles/main.css`:

- добавить стили для preview-кнопки, modal overlay и modal content;
- сохранить текущую сетку `.file-item` и mobile layout;
- не допустить переполнения имени файла и кнопок на мобильных экранах.

## API contract

Без изменений.

Существующие endpoint-ы остаются:

```txt
POST /api/v1/appeal-drafts/{id}/attachments
DELETE /api/v1/appeal-drafts/{id}/attachments/{attachment}
```

Новые endpoint-ы не добавляются.

## SEO impact

Нет влияния на SEO.

Страница `/appeal/new` относится к пользовательскому workflow, а не к индексируемому публичному контенту.

## Tests

Минимальные проверки:

- `pnpm lint` во frontend;
- ручная проверка `/appeal/new`:
  - JPG/PNG/WEBP после загрузки показывают миниатюру;
  - MP4/MOV не ломают карточку и открываются в preview, если поддерживаются браузером;
  - PDF открывается в preview modal;
  - удаление файла убирает preview и не оставляет битую карточку;
  - после перехода между шагами preview сохраняется в рамках текущей страницы;
  - после перезагрузки страницы для старых вложений остаётся текущая типовая карточка без локального preview.

## Acceptance criteria

- [ ] Для только что выбранных изображений список вложений показывает реальную миниатюру.
- [ ] Для файлов с локальным preview доступна кнопка "Просмотреть".
- [ ] Preview открывается в модальном окне и закрывается без потери данных формы.
- [ ] PDF не пытается отображаться как изображение.
- [ ] Удаление вложения очищает связанный object URL.
- [ ] При размонтировании страницы все object URLs освобождаются.
- [ ] На мобильной ширине карточки файлов не ломают сетку и текст не наезжает на кнопки.
- [ ] API-контракт и backend остаются без изменений.

## Postman

Postman collection обновлять не нужно, потому что API endpoint-ы и payload-ы не меняются.
