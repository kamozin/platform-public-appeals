# Переделать блок выбора направления на главной

## Context

На главной странице блок "Куда направить обращение или жалобу" уже содержит категории, описания и CTA. Текущий вариант выглядит слишком таблично: много одинаковых рамок, повторяющиеся кнопки и слабая визуальная иерархия делают блок тяжёлым и недостаточно аккуратным.

## Goal

Сделать новый визуальный вариант блока выбора направления:

- заменить сетку одинаковых карточек на интерфейс маршрутизатора;
- выделить самые частые темы отдельной зоной;
- показать остальные направления компактным списком;
- уменьшить визуальный шум от повторяющихся кнопок;
- сохранить быстрые ссылки на форму обращения.

## Non-goals

- Не менять Laravel API.
- Не добавлять новые endpoints.
- Не менять бизнес-логику обращений.
- Не редактировать `static/`.
- Не менять соседние блоки главной страницы.
- Не менять Postman collection.

## Backend changes

Backend не меняется.

## Frontend changes

Изменить:

- `frontend/app/components/home/HomeCategoriesPreview.vue`;
- `frontend/app/assets/styles/main.css`;
- `docs/development/frontend-home-page.md`.

Ожидаемые изменения:

- компонент делит категории на частые и остальные;
- частые темы показываются в крупной левой зоне выбора;
- остальные направления показываются компактным списком справа;
- мобильная версия остаётся одноколоночной и не ломает текст.

## API contract

API-контракт не меняется.

## SEO impact

Страница `/` остаётся SSR-страницей. Тексты категорий остаются в HTML и доступны поисковым системам без client-side JavaScript.

## Tests

Проверить frontend:

```bash
docker compose exec nuxt sh -lc "corepack pnpm typecheck"
docker compose exec nuxt sh -lc "corepack pnpm lint"
docker compose exec nuxt sh -lc "corepack pnpm build"
```

Дополнительно:

- SSR smoke главной страницы;
- визуальная проверка desktop/mobile;
- `git diff --check` по затронутым файлам.

## Acceptance criteria

- Блок выбора направления больше не выглядит как сетка одинаковых карточек.
- Частые направления визуально выделены и ведут на форму обращения.
- Остальные направления читаются компактно и не конкурируют с главными.
- На мобильных экранах список остаётся читаемым, без наложений и вылезания текста.
- `static/` не изменён.
- Документация на русском обновлена.

## Postman

Postman collection не обновляется, потому что API endpoints не добавляются и не меняются.
