# Усилить конверсионные блоки главной страницы

## Context

На главной странице Nuxt уже есть блоки статистики, категорий обращений и сценария "Как это работает". Текущая версия выглядит аккуратно, но слишком мелко и недостаточно продающе: все блоки имеют похожую визуальную силу, карточки категорий выглядят как вспомогательная навигация, а сценарий подачи обращения не завершает пользователя явным действием.

## Goal

Сделать верхние конверсионные блоки главной страницы заметнее и убедительнее:

- усилить статистику как social proof;
- сделать категории более кликабельными и понятными;
- сократить сценарий "Как это работает" до более крупного и продающего пути;
- добавить явные CTA к подаче обращения;
- сохранить SSR и текущую компонентную структуру Nuxt.

## Non-goals

- Не менять Laravel API.
- Не добавлять новые endpoints.
- Не менять бизнес-логику обращений.
- Не редактировать `static/`.
- Не переносить другие страницы.
- Не менять рекламный блок и ленту обращений.

## Backend changes

Backend не меняется.

## Frontend changes

Изменить:

- `frontend/app/components/home/HomeStatsStrip.vue`;
- `frontend/app/components/home/HomeCategoriesPreview.vue`;
- `frontend/app/components/home/HomeHowItWorks.vue`;
- `frontend/app/assets/styles/main.css`;
- документацию по главной странице в `docs/development/frontend-home-page.md`.

Ожидаемые изменения:

- статистика получает короткие подписи, поясняющий текст и кнопку подачи обращения;
- категории получают описания, бейджи популярных направлений и сильные hover/focus состояния;
- блок "Как это работает" превращается в 4 крупных шага с пояснениями и финальным CTA;
- адаптивность сохраняется для desktop/tablet/mobile.

## API contract

API-контракт не меняется.

## SEO impact

Страница `/` остаётся SSR-страницей. Основной `h1`, meta, canonical и Open Graph остаются в `frontend/app/pages/index.vue`.

Изменения улучшают видимый HTML-контент главной страницы, но не добавляют новые SEO endpoints.

## Tests

Проверить frontend:

```bash
pnpm typecheck
pnpm lint
pnpm build
```

Если локальная среда не готова, зафиксировать причину и выполнить доступные проверки.

Фактические проверки по выполненной задаче фиксируются в отчёте:

- `typecheck`, `lint`, `build` в Docker-сервисе `nuxt`;
- SSR smoke главной страницы;
- визуальная проверка desktop и mobile через headless Edge screenshots;
- `git diff --check` по затронутым файлам.

## Acceptance criteria

- Блок статистики визуально крупнее и содержит явный CTA.
- Категории выглядят как основное действие, а не мелкая иконная сетка.
- Блок "Как это работает" содержит 4 крупных шага, понятных без мелкого текста.
- На мобильных экранах карточки не ломают сетку и текст не вылезает из контейнеров.
- `static/` не изменён.
- Документация на русском обновлена.

## Postman

Postman collection не обновляется, потому что API endpoints не добавляются и не меняются.
