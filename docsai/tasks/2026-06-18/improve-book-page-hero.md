# Улучшение hero-блока страницы жалобной книги

## Context

На публичной SSR-странице `/book` первый экран выглядит несбалансированно: общий фон `content-hero` сильно высветляет фотографию, оставляет много пустого пространства, а обложка книги визуально уходит вправо и перетягивает внимание. Страница уже существует в Nuxt и не зависит от backend API.

## Goal

Сделать первый экран `/book` аккуратнее и плотнее:

- оформить hero как отдельную композицию страницы жалобной книги;
- уменьшить ощущение пустоты и убрать визуальный перекос вправо;
- сохранить SSR, текущий `h1`, смысл текста и изображение обложки;
- добавить понятные действия к связанным публичным сценариям.

## Non-goals

- Не менять backend API.
- Не подключать реальный реестр обращений.
- Не менять SEO-контракт, sitemap и robots.
- Не редактировать исходную папку `static/`.
- Не менять общую композицию остальных публичных страниц.

## Backend changes

Backend не меняется.

## Frontend changes

- Обновить `frontend/app/pages/book.vue`:
  - добавить CTA-ссылки из hero на `/appeals` и `/appeal/new`;
  - сохранить основной заголовок и описание страницы.
- Обновить `frontend/app/assets/styles/main.css`:
  - сделать для `.book-hero` отдельный фон и сетку;
  - ограничить ширину текста и размер обложки;
  - добавить адаптивные стили для CTA и мобильной версии.

## API contract

API-контракт не меняется.

## SEO impact

SEO не меняется: страница остается SSR, `title`, `description`, canonical и основной `h1` не меняются.

- [x] Визуальная проверка Edge headless, desktop viewport `1530x827`.
- [x] Визуальная проверка Edge headless, mobile viewport `390x844`.
- [x] `git diff --check` по затронутым файлам.
- [ ] `pnpm typecheck`, `pnpm lint`, `nuxt build` не выполнены из-за неконсистентного Node/pnpm окружения в доступном shell.

## Acceptance criteria

- Hero `/book` больше не выглядит как растянутая пустая плашка.
- Обложка книги не обрезается и не прижата к правому краю.
- Текстовый блок читается компактно и не перекрывает изображение.
- CTA-ссылки ведут на существующие Nuxt-маршруты.
- `static/` не редактировалась.

## Postman

Postman collection обновлять не нужно, потому что API endpoints и контракты не меняются.
