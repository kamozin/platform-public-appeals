# Сделать детальную страницу новости эффектнее

## Context

Детальная страница `/news/[slug]` уже получает данные из Laravel API на SSR, но визуально использует минимальную композицию: заголовок, обложку и текст. В `static/news-detail.html` и общих стилях уже есть более богатая новостная структура с мета-информацией, обложкой, статьёй и боковой колонкой.

## Goal

Сделать страницу новости более выразительной без изменения API-контракта: усилить первый экран, добавить мета-информацию, CTA, боковую сводку и аккуратную адаптивную верстку.

## Non-goals

- Не менять Laravel API.
- Не добавлять CMS или новые поля новости.
- Не править исходную папку `static/`.
- Не подключать дополнительные frontend-зависимости.

## Backend changes

Backend не меняется.

## Frontend changes

- Обновить `frontend/app/pages/news/[slug].vue`.
- Использовать существующие данные `NewsItemDto`.
- Добавить вычисляемое время чтения и боковую сводку.
- Доработать стили `news-detail-*` в `frontend/app/assets/styles/main.css`.
- Зафиксировать форматирование русских дат в московской таймзоне для стабильного SSR.

## API contract

Без изменений. Используется существующий `GET /api/v1/news/{slug}`.

## SEO impact

SSR и `usePageSeo()` сохраняются. Заголовок, description, canonical, robots и OG-изображение продолжают приходить из текущего DTO.

## Tests

- `pnpm typecheck`
- `pnpm lint`

## Acceptance criteria

- [x] Детальная страница новости визуально отличается от базовой content-страницы.
- [x] Первый экран содержит категорию, дату, время чтения, заголовок, описание и CTA.
- [x] Обложка новости остаётся реальным изображением из DTO с fallback.
- [x] Основной текст рендерится на SSR.
- [x] Дата публикации не сдвигается на предыдущий день на серверном рендеринге.
- [x] На мобильной ширине нет горизонтального переполнения и наложения текста.

## Postman

Postman collection не меняется, так как API-контракт не меняется.
