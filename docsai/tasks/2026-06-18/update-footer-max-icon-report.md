# Замена иконки MAX в футере

## Context

В публичном футере кнопка MAX использовала иконку из общего SVG-спрайта. В `frontend/public/assets/` уже есть отдельный файл `MAX.svg`, который должен использоваться как актуальная иконка сервиса.

## Goal

Заменить отображение иконки MAX в футере на файл `/assets/MAX.svg`, не меняя остальные социальные кнопки и не редактируя исходную папку `static/`.

## Changes

- В `frontend/app/components/layout/AppFooter.vue` для кнопки MAX добавлен путь к изображению `/assets/MAX.svg`.
- Для социальных кнопок добавлен небольшой тип `FooterSocialButton`.
- В шаблоне футера MAX теперь рендерится как декоративный `img`, а остальные иконки продолжают использовать `LayoutAppIcon`.
- В `frontend/app/assets/styles/main.css` добавлены размеры для изображения MAX внутри футера.

## Tests

- Проверен фрагмент `AppFooter.vue` после изменения.
- Проверен CSS-фрагмент социальных иконок.
- Выполнен `git diff --check` для затронутых frontend-файлов.

Полный `pnpm typecheck` не запускался: `pnpm` не установлен ни в текущем PowerShell PATH, ни внутри WSL. Запуск через Windows `corepack` завершился ошибкой подписи Corepack и проблемой UNC-пути.

## Postman

Не требуется: API endpoints не менялись.

## Acceptance criteria

- MAX в футере использует `/assets/MAX.svg`.
- VK и Одноклассники продолжают использовать прежние спрайтовые иконки.
- `static/` не изменялась.
- Изменение не затрагивает API-контракты.
