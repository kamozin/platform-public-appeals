# Добавить фирменную 404 страницу

## Context

Сейчас Nuxt использует стандартный экран ошибки для несуществующих маршрутов. Он не соответствует визуальной системе проекта: шапке, футеру, публичным карточкам, синей/красной палитре и смыслу платформы обращений граждан.

Для публичного сайта нужна понятная 404 страница, которая сохраняет доверительный тон проекта, помогает пользователю вернуться к полезным разделам и не попадает в индекс поисковых систем.

## Goal

Добавить кастомную Nuxt error page для статуса 404 в стилистике проекта «Рука добра».

Страница должна:

- использовать общий layout с шапкой и футером;
- показывать понятное сообщение о ненайденной странице;
- предлагать переходы на главную, список обращений и категории;
- иметь `noindex,nofollow`;
- не менять backend API.

## Non-goals

- Не менять Laravel backend.
- Не добавлять новые API endpoints.
- Не менять маршрутизацию существующих публичных страниц.
- Не редактировать `static/`.
- Не делать отдельный SPA-only fallback.
- Не обновлять Postman collection.

## Backend changes

Backend не меняется.

## Frontend changes

- Добавить `frontend/app/error.vue`.
- Для 404 показывать фирменный текст и действия.
- Для прочих ошибок использовать тот же экран с нейтральным сообщением.
- Добавить CSS-классы 404 страницы в `frontend/app/assets/styles/main.css`.
- Использовать существующие `LayoutAppIcon`, кнопки и цветовые переменные.

## API contract

API-контракт не меняется.

## SEO impact

- Error page должна выставлять `robots: noindex,nofollow`.
- Canonical для 404 не добавляется.
- Sitemap и robots routes не меняются.
- SSR-статус 404 должен сохраняться через Nuxt error page.

## Tests

- Запустить `pnpm lint`.
- Запустить `pnpm typecheck`.
- При необходимости запустить `pnpm build`.
- Локально проверить, что неизвестный маршрут показывает кастомный экран.

## Acceptance criteria

- [x] Несуществующий публичный URL показывает фирменную 404 страницу.
- [x] Страница визуально согласована с текущими публичными страницами.
- [x] Есть действия «На главную», «Смотреть обращения», «Выбрать категорию».
- [x] Текст не ломается на мобильных ширинах.
- [x] Для error page установлен `noindex,nofollow`.
- [x] Backend и `static/` не меняются.

## Postman

Обновление Postman collection не требуется, потому что API-контракт не меняется.
