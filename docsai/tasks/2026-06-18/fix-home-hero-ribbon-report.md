# Исправление нижней ленты главного баннера

## Context

На главной странице нижняя декоративная лента в hero-блоке выглядела как тяжёлая полноширинная полоса и визуально спорила с CTA-кнопками и точками слайдера.

## Goal

Сделать нижний декоративный элемент менее навязчивым, убрать ощущение отдельной прямоугольной плашки и сохранить читаемость действий в hero-блоке.

## Non-goals

- Не менять исходную папку `static/`.
- Не менять структуру hero-компонента и тексты слайдов.
- Не менять API, backend и SEO-контракт.

## Backend changes

Backend не менялся.

## Frontend changes

- В `frontend/app/assets/styles/main.css` изменён `hero::after`: лента больше не растягивается на всю ширину баннера, стала компактным нижним акцентом.
- Для `.hero-content` повышен stacking context, чтобы контент и CTA были выше декоративного слоя.
- Добавлены адаптивные значения ширины, высоты и прозрачности ленты для tablet/mobile viewport-ов.

## API contract

API-контракт не менялся.

## SEO impact

SEO не затронуто: SSR, meta, sitemap и robots не менялись.

## Tests

- `docker compose exec -T nuxt pnpm lint`
- `curl.exe -I -k https://rukadobra.localhost/`
- Дополнительно выполнена базовая проверка баланса CSS-скобок через Node.

## Acceptance criteria

- [x] Нижняя лента hero-блока не выглядит как полноширинная прямоугольная плашка.
- [x] CTA-кнопки остаются визуально читаемыми и не перекрываются декоративным слоем.
- [x] На мобильных ширинах лента приглушена и не конфликтует с полноширинными кнопками.
- [x] `static/` не редактировалась.

## Postman

Postman collection не требуется обновлять, потому что API endpoints и контракт не менялись.
